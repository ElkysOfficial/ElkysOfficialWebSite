# Cases Images

Esta pasta contém as imagens dos cases de sucesso.

## Requisitos das Imagens

Para melhor aparência e consistência visual:

- **Formato:** JPG ou PNG
- **Dimensões recomendadas:** 1600x1000px (aspect ratio 16:10)
- **Tamanho máximo:** 800KB por imagem
- **Qualidade:** Alta resolução
- **Conteúdo:** Screenshots, mockups ou fotos do projeto

## Nomenclatura dos Arquivos

As imagens devem ser nomeadas como:
- `case-1.jpg` - Hapvida
- `case-2.jpg` - God Of Barber
- `case-3.jpg` - Antônio Oliveira Advogados
- `case-4.jpg` - 1Um Print Comunicação
- `case-5.jpg` - PlansCoop
- `case-6.jpg` - AK Produções

## Sugestões de Conteúdo para Imagens

1. **Screenshots da interface** - Capturas das telas principais do projeto
2. **Mockups de dispositivos** - Imagens do projeto em smartphones/tablets/desktop
3. **Design antes/depois** - Comparativos de transformação
4. **Dashboards e gráficos** - Visualização de resultados
5. **Hero images** - Imagens promocionais do projeto

## Atualizar Informações dos Cases

Para atualizar as informações dos cases, edite o arquivo:
`src/components/CaseStudies.tsx`

Altere o array `caseStudies` com as informações corretas:
- `title`: Título do case
- `client`: Nome do cliente
- `category`: Categoria/setor
- `description`: Descrição breve do projeto
- `image`: Caminho da imagem (ex: `/imgs/cases/case-1.jpg`)
- `results`: Array com métricas e resultados
- `tags`: Array com tecnologias/serviços utilizados
- `link`: URL para página detalhada (opcional)

## Dicas de Fotografia/Design

- Use imagens profissionais e de alta qualidade
- Mantenha consistência visual entre os cases
- Evite imagens com muito texto
- Prefira imagens que mostrem a interface do produto
- Use cores que complementem a paleta da Elys (roxo/azul)
